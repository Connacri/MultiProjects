# Planning Source of Truth

The new planning engine does not use legacy planning assignments as generation input.

## Generation inputs

A new planning is generated exclusively from:

1. current staff members;
2. current team/group organization;
3. current leave/absence periods;
4. versioned rotation configuration;
5. optional continuity state from a planning previously published by the new engine.

Historical assignments from the old planning system are not read, imported, or used to infer rotation.

## First planning with the new engine

The first planning generated by the new engine starts from the configured reference date and reference phase.

After publication, the engine stores its own continuity state. Future months can continue from that state.

## Published planning

A published planning is an output/snapshot, not a generation input for staff-level assignments. Only its explicit rotation state may be consumed for continuity.

```text
Current Staff + Leaves + Rotation Config
                 │
                 ├───────────────┐
                 │               │
                 ▼               ▼
          Rotation Engine   Availability Rules
                 │               │
                 └───────┬───────┘
                         ▼
                  New Planning
                         │
                         ▼
                      Publish
                         │
             ┌───────────┴───────────┐
             ▼                       ▼
     Immutable Snapshot      Rotation State
```

## Explicit non-goals

The new generator must never:

- read old `ActiviteJour` records to reconstruct a rotation;
- infer the new rotation from old monthly assignments;
- silently import old statuses into a new planning;
- regenerate a historical published snapshot because the current rotation configuration changed.
